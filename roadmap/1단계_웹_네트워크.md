## 📅 1주차: 브라우저 동작 원리 이해 (일일 계획)

### **Day 1 – 렌더링 기본 흐름 이해**

- **이론 (40분)**
    
    - 브라우저가 HTML → DOM, CSS → CSSOM을 만드는 과정
        
    - Render Tree, Layout, Paint, Composite 단계 정리
        
- **실습 (50분)**
    
    - 간단한 HTML+CSS 작성 후 DevTools Performance 탭으로 렌더링 확인
        
    - `will-change`, `transform`, `opacity` 속성이 렌더링에 미치는 영향 테스트
        
- **정리 (30분)**
    
    - DOM & CSSOM → Render Tree 흐름을 다이어그램으로 그려보기
        

---

### **Day 2 – Reflow vs Repaint**

- **이론 (40분)**
    
    - Reflow: 레이아웃 계산 재실행
        
    - Repaint: 픽셀 다시 그리기
        
    - 어떤 CSS 속성이 Reflow를 일으키는지 정리
        
- **실습 (50분)**
    
    - `width/height` 변경 시 Reflow 발생 여부 확인
        
    - `color`만 바꿀 때는 Repaint만 발생함을 DevTools로 확인
        
- **정리 (30분)**
    
    - “Reflow vs Repaint 비교표” 작성
        

---

### **Day 3 – 이벤트 루프 기본**

- **이론 (40분)**
    
    - Call Stack, Task Queue, Microtask Queue 구조
        
    - setTimeout, setInterval, Promise, async/await의 실행 순서
        
- **실습 (50분)**
    
    - `console.log`, `setTimeout`, `Promise.resolve`를 섞은 코드 실행 → 실행 순서 기록
        
    - DevTools Performance 탭에서 Call Stack 시각화
        
- **정리 (30분)**
    
    - Event Loop 순서를 글로 설명할 수 있게 정리
        

---

### **Day 4 – 이벤트 루프 심화**

- **이론 (40분)**
    
    - MacroTask vs MicroTask (setTimeout vs Promise vs MutationObserver)
        
    - 브라우저 렌더링 타이밍과의 관계
        
- **실습 (50분)**
    
    - async/await 안에서 setTimeout과 Promise를 섞어 실행 순서 확인
        
    - MutationObserver 활용 실험
        
- **정리 (30분)**
    
    - “MicroTask > MacroTask” 실행 순서를 다이어그램으로 정리
        

---

### **Day 5 – 종합 실습**

- **실습 (1시간)**
    
    - 렌더링 최적화: Reflow 최소화 코드 작성 (DOM 조작 시 `documentFragment`, `classList.toggle` 활용)
        
    - 이벤트 루프 시뮬레이션 문제 풀기 (실행 순서 예측 후 확인)
        
- **정리 (1시간)**

    - 이번 주 학습 키워드(렌더링 파이프라인, Reflow/Repaint, Event Loop) 요약
